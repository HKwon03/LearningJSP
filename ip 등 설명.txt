
IP Address : 목적지 시스템을 식별하는 주소 체계 (v4 : 32bit(약 42억 9천개), v6 : 128bit(340 언데실리온(10^36))
	8bit.8bit.8bit.8bit
	8bit = 옥텟
	Network ID + Host ID	<== Subnet Mask가 Network ID, Host ID를 식별하는 식별자
	A, B, C 만 목적지를 식별할 때 사용됨.
	D Class : 멀티캐스트(Multicast) 데이터를 전송시에 사용, <<1 : 그룹 통신>>
	E Class : 예약, 군사용, 만약의 사태에 대비하기 위해서 남겨둔 IP 대역

A Class	N.H.H.H
(2진수)	00000000.00000000.00000000.00000000	1.0.0.0	(10진수)
	01111111.11111111.11111111.11111111	126.255.255.255
	Network 수 : 2^7 -2 = 126
	Host 수	  : 2^24 -2 = 16,777,214
B Class	N.N.H.H
	10000000.00000000.00000000.00000000	128.0.0.0
	10111111.11111111.11111111.11111111	191.255.255.255
	Network 수 : 2^14 = 16,384
	Host 수	  : 2^16 - 2 = 65,534

C Class	N.N.N.H
	11000000.00000000.00000000.00000000	192.0.0.0
	11011111.11111111.11111111.11111111	223.255.255.255
	Network 수 : 2^21 = 2,097,152
	Host 수 	  : 2^8 - 2 = 254

	11100000.00000000.00000000.00000000	224.0.0.0
	11101111.11111111.11111111.11111111	239.255.255.255

E Class : 예약, 군사적.

특수하게 사용되는 IP 범위 :
 A class : 
	0 Network	: network를 식별할때 사용 
	127 Network	: Loop Back IP, 자신의 컴퓨터를 뜻한다. 랜카드까지 갔다가 다시 자신의 시스템으로 돌아온다.
			: 개발 목적, 테스트 목적으로 사용하는 IP
			: Server를 구동하기 위해서는 IP

Network ID : Network 를 뜻한다. (각각의 회사(하나의 회사는 동일한(하나의) Network ID를 쓴다)), IP Address 와 Sub NetMask 를 AND 연산해서 Network ID를 식별
	IANA : 세계의 IP를 관리하는 단체 ==> KRNIC ==> ISP(Internet Service Provider) : KT, SK, LG
	(우편번호 <== 동까지 식별)

Host ID : Host(네트워크 내부에 존재하는 장치(라우터, 스위치, 프린터, 컴퓨터))
	(번지 식별)
 - Host 수는 항상 2개를 뺀다.
     - Host의 모든 비트가 0인 경우 : Network ID 자체를 식별할 때 사용되는 식별자
     - Host의 모든 비트가 1인 경우 : Broad cast 데이터를 보낼때 사용

데이터 전송방식
	- Unicast		: 1 : 1 통신방법
	- Multicast	: 1 : 그룹(d class) 통신방법
	- Broadcast	: 1 : 모든 (라우터 장비까지)

TCP/IP
  IP Address : 목적지를 식별
  Subnet Mask : Network ID 와  Host ID를 식별해주는 식별자. 1이 들어있는 비트까지 : Network ID, 0이 들어있는 비트 : Host ID
  Gateway : 외부로 데이터를 보낼때 Gateway(라우터)로 전송, 외부 패킷이 내부로 전송될 때 Gateway를 통해서 전송.
	Gateway IP를 넣지 않을 경우 외부 시스템과 통신할 수 없다.
	내부 시스템에서만 통신하고 외부 시스템과는 통신하지 못하도록 특정 서버는 Gateway IP 없이 서버를 세팅
  DNS IP : FQDN 이름을 IP Address로 변환(www.google.co.kr ==> 142.250.204.131)
				 (www.naver.com ==> 223.130.200.107)

문제 1] IP Address 와 Subnet Mask 정보만 있으면 다음정보를 알 수 있다.
     192.168.100.100 /17    CIDR   , 십진수 Subnet Mask 표시, Host 수, Network ID, Broadcast IP

	192.168.100.100		1100 0000. 1010 1000. 0110 0100. 0110 0100
	255.255.128.0		1111 1111. 1111 1111. 1000 0000. 0000 0000	
=========================================================
AND	192.168.0.0		1100 0000. 1010 1000. 0000 0000. 0000 0000

	Network ID : 192.168.0.0		<== IP와 Subnet Mask를 AND 연산해서 Network ID를 찾는다.
	Broadcast IP : 192.168.127.255		<== Network ID를 식별하면 Host의 모든 IP가 1이 들어가는 IP
		1100 0000. 1010 1000. 0111 1111. 1111 1111

	IP를 부여할 수 있는 범위 : 
		192.168.0.2 ~ 192.168.127.254
	Host 수 : 2^15 -2 = 32,766
	Gateway IP : 192.168.0.1	<== 부여할 수 있는 IP 범위의 제일 처음 IP나 제일 마지막 IP를 일반적으로 Gateway IP로 세팅.

Subnetting : IANA(IP 관리하는 단체) --> KRNIC --> ISP(KT, SK, LG...), 보안, 네트워크 성능
	ISP 에서 Network ID 를 부여해 준다. Network ID를 작은 네트워크로 분할해서 사용(라우터), 하나의 네트워크에 많은 Host가 있으면 네트워크 성능이 저하

	192.168.100.0/24 네트워크 ID를 부여받았다. 2개의 네트워크로 분할.
	1100 0000. 1010 1000. 0110 0100. 0000 0000	192.168.100.0/25
	1100 0000. 1010 1000. 0110 0100. 1000 0000	192.168.100.128/25

	192.168.100.0/24 네트워크 ID를 부여받았다. 4개의 네트워크로 분할.
	1100 0000. 1010 1000. 0110 0100. 0000 0000	192.168.100.0/26
	1100 0000. 1010 1000. 0110 0100. 0100 0000	192.168.100.64/26
	1100 0000. 1010 1000. 0110 0100. 1000 0000	192.168.100.128/26

공인 IP(Public) Address : 서버를 구현할 때 서버는 공인 IP를 가져야 한다.

비공인(사설: Private) IP : 보안을 위해서, 적은 공인 IP로 많은 시스템을 인터넷 연결.

	A Class : 10.x.x.x
	B Class : 172.16.x.x ~ 172.31.x.x
	C Class : 192.168.x.x
    사설 IP Address가 인터넷을 사용하기 위해서는 NAT(Network Address Translation)
	NAT : 패킷의 소스 Address를 공인 IP로 변환해서 내보낸다.

TCP/IP : 신뢰성 있는 데이터 전송시에 사용. http, ftp, smtp, ssh, telnet, pop, ...
	데이터가 오류없이 잘 전송되었는지 3번 확인

UDP/IP : 신뢰성 없는 데이터 전송시에 사용. 실시간 동영상 스트리밍 서비스할 때 주로 사용.
	데이터를 전송하면 끝난다. <데이터 전송이 빠르다>

Port : 데이터가 전송되는 통로(1 ~ 65,535)
   1 ~ 1024 : 잘 알려진 포트
	TCP 80 : Web
	TCP 21 : FTP
	TCP 22 : SSH
	TCP 23: Telnet
	TCP 110 : POP(메일 수신)
	TCP 25 : SMTP(메일 송신)
	TCP 53 : DNS(영역 전송 : Primary DNS의 DB를 Secondary DNS로 복제)
	UDP 53 : DNS(이름풀이 : www.naver.com ==> IP Address)
